## üßë‚Äçüíª How to Contribute

### 1. Fork the Repository

Create your own copy of the project.

### 2. Clone Your Fork

```bash
git clone <your-fork-url>
cd silver-talon
```

### 3. Create a New Branch

Always create a new branch before starting your work.  
Use a **clear and meaningful branch name** that reflects your change.

```bash
git checkout -b feature/short-description
```

### 4. Make Your Changes

While working on your changes:

- Write clean and readable code
- Follow the existing project structure
- Add comments where logic may not be obvious
- Focus on clarity and maintainability.

### 5. Commit Your Work

Keep commit messages short, clear, and meaningful.

```bash
git commit -m "Add table availability check logic"
```

Avoid vague messages like _update code_ or _fix bug._

### 6. Push and Open a Pull Request

Push your branch to your fork:

```bash
git push origin feature/short-description
```

When opening a Pull Request:

- Clearly explain what you changed
- Mention why the change is needed
- Add screenshots if the change affects the UI

## üß™ Code Quality Expectations

Please ensure:

- Meaningful variable and function names
- No hardcoded values
- Components are small and focused
- SQL data matches what is shown in the UI
- Changes are tested before submission

## üìù Pull Request Checklist

Before submitting your PR, confirm:

- Code is written and fully understood by you
- No direct AI copy-paste
- Code runs without errors
- Changes are focused and related
- PR description is clear and complete
